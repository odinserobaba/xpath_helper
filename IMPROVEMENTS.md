# Возможные улучшения XPath Helper Pro

Идеи для развития расширения: новые функции, удобство и стабильность.

---

## Удобство и интерфейс

- **Переименование сценариев** — поле «Название сценария» в списке, сохранение имени в JSON и в storage, отображение в заголовке вкладки «Список».
- **Несколько сохранённых сценариев** — переключение между сценариями (выпадающий список или вкладки), отдельное хранилище по ключу (например, по имени).
- **Группировка шагов** — блоки «Группа 1», «Группа 2» с возможностью сворачивания и выполнения только выбранной группы.
- **Поиск по списку шагов** — фильтр по XPath или по типу действия.
- **Отмена выполнения** — кнопка «Стоп» во время прогона списка, проверка флага в content script между шагами.
- **Подсказки (tooltips)** — краткое описание к кнопкам и полям (например, что такое «Ожидание селектора»).

---

## Генерация XPath и инспектор

- **Режим «только уникальные»** — переключатель в настройках: показывать в списке только уникальные XPath.
- **Копирование всех XPath одной кнопкой** — уже есть «Все уникальные»; можно добавить «Все (уникальные + неуникальные)» в одном формате.
- **История последних элементов** — список последних 5–10 наведённых элементов с быстрым повторным выбором XPath.
- **Поддержка других селекторов** — генерация не только XPath, но и CSS-селекторов (для сравнения и выбора).
- **Валидация XPath при вводе вручную** — в модалке шага: проверка синтаксиса и количества совпадений на текущей странице перед сохранением.

---

## Выполнение и автотесты

- **Задержка между шагами** — глобальная настройка «Пауза между шагами (мс)» для стабильности на медленных страницах.
- **Повтор при ошибке** — опция «Повторить шаг N раз при ошибке» с таймаутом.
- **Снимки (скриншоты)** — опциональное создание скриншота после каждого шага или только при ошибке (через `chrome.tabs.captureVisibleTab`).
- **Лог выполнения** — окно или секция «Лог»: какой шаг выполнен, успех/ошибка, время; экспорт лога в текст/JSON.
- **Шаблоны автотестов** — экспорт не только «сырого» JSON, но и заготовок под популярные фреймворки (Playwright, Cypress, Selenium и т.п.) на основе того же формата шагов.
- **Запуск по расписанию** — напоминание или черновая идея: напоминать о необходимости прогнать сценарий (без полноценного фонового запуска, чтобы не усложнять политику расширения).

---

## Загрузка файлов и данные

- **Несколько файлов в одном шаге** — для `input[type=file][multiple]`: массив файлов (fileName + fileContentBase64) в params.
- **Ограничение размера файла в JSON** — предупреждение при прикреплении большого файла (например, > 1 MB) и опция «не включать содержимое в экспорт» (только fileName).
- **Хранение файлов отдельно** — большие файлы хранить по id в `chrome.storage.local` или IndexedDB, в шаге только ссылка на id (чтобы не дублировать в каждом сценарии).

## Надёжность и производительность

- **Восстановление после «Extension context invalidated»** — при перезагрузке расширения показывать в панели сообщение «Расширение обновлено, перезагрузите вкладку» и кнопку обновления.
- **Обработка SPA-навигации** — при смене URL не сбрасывать список шагов; опция «привязать сценарий к URL» (предупреждение при запуске на другом домене).
- **Ограничение числа шагов** — предупреждение при очень длинном списке (например, > 100 шагов) и возможность разбить на несколько сценариев.
- **Дебаунс и таймауты** — уже есть debounce наведения и таймаут селектора; при необходимости вынести константы (макс. размер JSON, макс. размер base64) в настройки или в manifest.

---

## Доступность и локализация

- **Полная поддержка screen readers** — aria-labels, роли, живые регионы для статусов выполнения (уже частично есть).
- **Локализация** — вынос строк в `_locales/ru/messages.json` (и при необходимости en), переключение языка по умолчанию из manifest.

---

## Безопасность и политика

- **Минимальные права** — возможность работать только на выбранных доменах (список в настройках) вместо `<all_urls>`, с предупреждением при открытии панели на неразрешённом сайте.
- **Предупреждение при импорте** — при загрузке JSON показывать превью (количество шагов, типы действий) и кнопки «Заменить список» / «Добавить к текущему» / «Отмена».

---

При внедрении улучшений удобно обновлять версию в `manifest.json` и при необходимости — секцию «Возможности» в README.
