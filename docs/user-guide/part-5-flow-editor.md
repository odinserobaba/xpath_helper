# Часть 5. Flow Editor (блок-схема)

Цель: редактировать сценарий как визуальную последовательность в стиле блок-схемы.

## Сценарий как файл (создать / имя / клон / удалить)

В левой колонке Flow Editor:

- **Новый…** — запросит имя и создаст новый JSON через API (`POST /api/scenarios`) с шагами **«Начало»** и **«Конец»**.
- **Клонировать…** — копия текущего сценария под новым именем (новый файл с новым id в имени).
- **Удалить** — удаляет файл сценария с диска (`DELETE`), с подтверждением.
- **Имя сценария (name)**, **smoke**, **labels**, **история** и **группы** — в модальном окне (**ПКМ по пункту списка сценария** → «Настройки сценария и групп»). Сохранение — **💾 Сохранить** в левой колонке. Имя файла (`{id}.json`) не меняется.

## Где открыть

1. Запустите раннер (`./run.sh`)
2. Откройте `http://127.0.0.1:8000`
3. Нажмите **Открыть Flow Editor**

## Запуск прогона из редактора

В левой колонке есть **▶ Запустить**: сначала выполняется **сохранение** сценария на диск (как **💾 Сохранить**), затем вызывается тот же **`POST /api/runs`**, что и на главной странице Web Runner. Открывается **live-страница прогона** в новой вкладке.

Параметры запуска (**startUrl**, **headless**, **viewport**, CDP и т.д.) берутся из **`runnerSettings`** в JSON сценария (их удобно задать на главной Web Runner кнопкой сохранения параметров сценария) либо остаются значениями по умолчанию сервера. Если задан **`XPATH_RUNNER_TOKEN`**, токен подставляется в страницу редактора автоматически (как для остальных запросов к API).

### Подсветка текущего шага на схеме (во время прогона)

В левой колонке блок **«Живая подсветка шага»**: укажите **runId** и нажмите **Подключить** — редактор подписывается на **`GET /api/runs/{runId}/events`** (тот же поток событий, что на live-странице прогона). Узел с тем же **`stepRef`**, что и событие **`step_start`**, подсвечивается на канве; после **`step_done`** подсветка снимается (после **`run_done`** показывается сообщение о завершении).

После **▶ Запустить** подписка на новый прогон включается **автоматически** (runId подставляется в поле). Можно открыть редактор сразу с run: **`/flow-editor?run=<runId>`**. При включённом **`XPATH_RUNNER_TOKEN`** токен для потока событий передаётся в **`?token=`** (браузерный `EventSource` не умеет заголовки).

Кнопка **⏹ Стоп** в том же блоке вызывает **`POST /api/runs/{runId}/stop`**: движок завершает прогон **после текущего шага** (между шагами; не обрывает середину действия Playwright). То же есть на **live-странице** прогона.

**ПКМ по шагу** на схеме (не по рамке группы): **«Настройки шага…»**, **«Запустить с шага #N»** (если доступно), **«Копировать»**. **ПКМ по примитиву разметки**: **«Настройки примитива…»** (в т.ч. картинка: URL, objectFit, прозрачность) и **«Копировать»**. **ПКМ по пунктирной связи разметки (decorate)**: **«Подпись и цвета…»**. Убедитесь, что **startUrl** и страница соответствуют точке входа при запуске с середины.

Интерфейс **двух колонок** (список слева, канва справа); прокрутка внутри **`panel-scroll`**, общий скролл страницы отключён на широком экране.

## Сборка (один раз)

Редактор грузится с **вашего** сервера (`/static/…`), без CDN. После `git pull` при необходимости пересоберите:

```bash
cd web-runner/flow-editor-src && npm ci && npm run build
```

Команда `web-runner/install.sh` уже вызывает эту сборку, если установлен Node.js.

## Шаг «Начало» (start)

В сценарии должен быть **хотя бы один** шаг с действием **«Начало»** (`action: "start"`). С него web-runner и расширение начинают выполнение (шаги *выше* в списке не выполняются). Опционально задайте подпись в `params.message` — она попадёт в лог.

При сохранении из Flow Editor без блока «Начало» сохранение будет отклонено с подсказкой.

При **Сохранить** блоки **`start` всегда записываются в JSON перед остальными**, а блоки **`end` — после всех остальных** (чтобы порядок шагов совпадал с точкой входа и явным завершением).

Старые JSON без `start` в web-runner по-прежнему запускаются **с первого шага**, в лог пишется предупреждение.

## Шаг «Конец» (end)

Блок **`end`** останавливает прогон сценария (web-runner и расширение выходят из цикла шагов после него). Удобно вести ветку **«Нет»** или отдельный путь в **«Конец»**, чтобы не выполнять дальнейшие шаги. Подпись — в `params.message`. XPath для `start`/`end` не задаётся (в JSON — `—`).

## Сетка на канве

Узлы **привязываются к сетке 16×16 px** при перетаскивании (как фоновая сетка фона канвы).

## Выделение нескольких блоков (lasso)

- На **пустом месте** канвы зажмите **левую кнопку мыши** и протяните **прямоугольник** — попавшие в него узлы выделятся.
- **Shift + клик** по узлу **добавляет** его к текущему выделению.
- **Панорама** канвы: **Space + левая кнопка** и перетаскивание, либо **средняя** или **правая** кнопка мыши (ЛКМ на пустом месте занята рамкой выделения).

Кнопки выравнивания на панели канвы:

- **📐 Сетка (все)** — расставить **все** узлы сеткой **2×N** (два столбца — больше горизонтального зазора, связи реже визуально слипаются) в порядке исполнения (**start** → по `stepRef` → **end**). Связи по умолчанию рисуются **кривыми Безье** (`simplebezier`), а не ортогональными «ломанными».
- **📐 Сетка (выделенные)** — то же для **только выделенных** (от левого верхнего угла их группы).
- **↔ По next** — слои **слева направо** по рёбрам типа **next** (от блока **Начало** или узлов без входящего `next`).

При нескольких выбранных узлах **🗑 Удалить** удаляет все выделенные. **📋 Копировать** на панели — при одном выбранном узле; дублирование также из **ПКМ** по шагу или примитиву.

## Что можно в MVP

- Открыть любой сохраненный сценарий из списка
- Редактировать узлы и связи
- Добавлять комментарий к узлу
- Создавать визуальные группы узлов
- Сохранять обратно в тот же JSON сценария

## Комментарии к узлу

1. Кликните узел в канве → **ПКМ** → **«Настройки шага…»** (или двойной путь через панель инструментов при необходимости).
2. Заполните поле **Комментарий** в модальном окне.
3. Нажмите **💾 Сохранить** сценарий.

Комментарий сохраняется в `flow.nodes[].comment`.

## Разметка канвы (рамка, овал, текст)

На панели над схемой кнопки **▭ Рамка**, **○ Овал**, **T Текст**, **🖼 Картинка** добавляют элементы **только для оформления**. Они **не являются шагами** сценария и **не выполняются** раннером.

- Позицию можно менять перетаскиванием; размер, цвета, подпись, для картинки — **URL / objectFit / прозрачность**: **ПКМ по примитиву** → **«Настройки примитива…»**. Примитивы рисуются **под** шагами: при перекрытии **выбирается шаг**, а не разметка.
- К **примитивам нельзя подключить** стрелки шагов (связи строятся только между блоками сценария).
- Автосетки (**📐 Сетка**, **↔ По next**) двигают **только шаги**; разметка остаётся на месте (кроме общего пересчёта рамок групп при перетаскивании).
- При сохранении разметка попадает в **`flow.annotations`** (в т.ч. `kind: "image"` с `imageUrl`, `objectFit`, `imageOpacity`). Старые сценарии без этого поля открываются как раньше.
- **Визуальные стрелки** от примитивов: **`kind: "decorate"`** в **`flow.edges`**, подпись и цвета — **ПКМ по стрелке** → **«Подпись и цвета…»**. Раннер и ветвления **игнорируют** эти рёбра.
- **Добавление примитивов:** кнопки **▭ / ○ / T / 🖼** включают **режим рисования** — протащите мышью на пустом месте канвы (**Esc** — выход). Палитра линии/заливки — под панелью (для фигур). Панорама в режиме рисования — **средняя/правая** кнопка.

## Визуальные группы

1. **ПКМ по сценарию в списке** → в модалке **+ Группа**, название и цвет
2. В **настройках шага** (модалка) — выбор группы и **«В группу»**
3. Нажмите **💾 Сохранить**

Группы сохраняются в `flow.groups[]` (только список `nodeIds`, без отдельных узлов в JSON).

На канве вокруг членов группы рисуется **полупрозрачная рамка** (пунктир цвета группы) с **заметным внешним отступом** от блоков: она имеет **`z-index` ниже шагов и связей**, не перехватывает клики по блокам (`pointer-events: none`) и **сама пересчитывается** при перетаскивании шагов и при изменении состава группы. Так блоки и линии остаются читаемыми поверх фона группы.

## Важно про ветвления (branch)

Шаг с действием **branch** на канве отображается **ромбом**. У него два выхода:

- **Зелёная** точка справа — переход **«Да»** (сохраняется как `nextStep` / `nextId`)
- **Красная** снизу — **«Нет»** (`nextElseStep` / `nextElse`)

Подписи **Да** / **Нет** видны на самом ромбе (даже если линии ещё не проведены). Новый шаг (**+ Шаг** / копия) **не создаёт** автоматических связей — их нужно провести вручную. Связь можно **выделить** кликом (подсветится).

При прогоне в **web-runner** ветвление уже выполняется: по resultado условия выбирается целевой шаг из `nextStep` / `nextElseStep` или из строкового id узла вида `step-N-...` (номер **N** совпадает с полем `step` в JSON).

Если в JSON уже заданы эти переходы, но в `flow` нет рёбер — редактор **достроит** их при открытии сценария.

При сохранении редактор синхронизирует переходы branch в двух форматах:

- `params.nextStep/nextElseStep` — для исполнения раннером
- `params.nextId/nextElseId` — для визуального редактора

Это нужно для обратной совместимости со старыми сценариями.

## QA-поля, smoke, поиск, история, валидация

- **`note`**, **`ticket`**, **`qaStatus`** (черновик / стабильный / flaky) — в панели свойств шага; попадают в JSON шага и в live-run.
- **`smoke`** и **`labels`** (JSON) — под именем сценария; `labels` хранятся в корне сценария.
- **Поиск по канве** — поле над схемой подсвечивает узлы по названию, действию, XPath, тегам, note, тикету и тексту разметки (`flow.annotations`).
- **Ctrl+S** — сохранить сценарий.
- **Версии `.history`** — выпадающий список снимков и кнопка **Восстановить** (текущий файл перед этим тоже попадёт в историю).
- При сохранении сервер проверяет сценарий: при ошибках сохранение отклоняется; предупреждения показываются в тосте после успешного сохранения.
