# Часть 4. Примеры готовых сценариев

Ниже короткие примеры, которые можно повторить вручную.

## Пример 1. Smoke-проверка главной страницы

Что делает:

1. Открывает главную страницу
2. Проверяет видимость кнопки входа
3. Проверяет заголовок страницы

Подходит для быстрой проверки "сайт жив".

## Пример 2. Логин успешен

Что делает:

1. Открывает страницу логина
2. Вводит email/пароль
3. Нажимает "Войти"
4. Проверяет, что есть элемент личного кабинета

Полезно добавить `tags`: `smoke`, `auth`.

## Пример 3. Проверка формы с датой

Что делает:

1. Открывает форму
2. Использует `set_date` для `input[type=date]`
3. Отправляет форму
4. Проверяет сообщение об успешном сохранении

Этот пример нужен, чтобы избежать ошибок формата даты.

## Пример структуры шага (упрощенно)

```json
{
  "action": "click",
  "selector": "//button[contains(., 'Войти')]",
  "title": "Нажать кнопку Войти",
  "tags": ["smoke", "auth"]
}
```

## Хорошие практики

- Давайте шагам понятные `title`
- Добавляйте `tags` для группировки запусков
- Для нестабильных элементов используйте fallback selectors
- После каждого изменения запускайте сценарий повторно
