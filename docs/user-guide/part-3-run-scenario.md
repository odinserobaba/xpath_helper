# Часть 3. Запуск сценария в Web Runner

Цель: запустить сценарий из интерфейса и увидеть результат.

## Шаг 1. Выберите сценарий

1. Откройте `http://127.0.0.1:8000`
2. В поле **Сценарий** выберите сохраненный JSON
3. Укажите **Start URL**

## Шаг 2. Запустите

Нажмите **Run**.  
После старта откроется страница вида `/runs/<runId>` с live-статусом.

## Шаг 3. Проверьте результат

- `PASS` — все шаги успешно выполнены
- `FAIL` — на одном из шагов ошибка (смотрите лог и отчет)

## Где смотреть артефакты

- Логи: `tests/logs/`
- Выходные данные раннера: `web-runner/outputs/`

## Лог выполнения и скриншоты

В текстовом логе прогона пишутся:

- для **navigate** — целевой URL и строка **«Страница после перехода»** (фактический URL);
- для **ветвления (branch)** — результат условия (Да/Нет), цель перехода и номер шага, куда перешли;
- для **ошибки** — URL на момент сбоя;
- для успешных шагов с селектором — короткая строка про использованный XPath (при необходимости).

В **report.json** у каждого шага:

- **`screenshot`** — основной снимок страницы после шага (как правило совпадает с `after` при успехе; при падении шага — кадр ошибки);
- **`before` / `after`** — полный экран до и после действия;
- **`urlAfter`** — URL страницы после шага;
- для **branch** дополнительно **`branchResult`**, **`jumpTo`**, **`branchTarget`**.

В **report.html** есть колонки «URL после» и «Скрин шага» со ссылками на файлы в `.../screenshots/`.

## Типичный поток работы

`записал -> сохранил -> запустил -> посмотрел отчет -> поправил сценарий`

## Отдельный визуальный редактор

Если удобнее редактировать сценарий как блок-схему, откройте **Flow Editor** на главной странице раннера.
Подробности: [Часть 5. Flow Editor](./part-5-flow-editor.md)

## Дополнительно (QA-режим)

- **Старт с шага** (`startStep`) и **окружение** (`environment`) — в форме запуска и в `runnerSettings` сценария; при старте с середины задайте корректный **Start URL**.
- **Метки** (`labels`) и **smoke** — в Flow Editor; в запуск уходит поле `labels` из списка сценариев.
- **Сравнение прогонов** — страница `/runs/compare?a=<runId>&b=<runId>` или API `GET /api/runs/compare?left=&right=`.
- **Метрики по сценарию** — при выборе сценария на главной показывается краткий блок (последние прогоны, «flaky»-шаги); API `GET /api/scenarios/{id}/insights`.
- **Отладка с паузами** — в `runnerSettings` задайте `debugBreakpoints`: массив номеров шагов (`step`); на паузе на live-странице нажмите **Продолжить отладку** или `POST /api/runs/{runId}/debug-continue`.
- **Консоль браузера** — флаг `captureConsole` в теле `POST /api/runs` или в `runnerSettings` → файл `console.log` в каталоге прогона.
- **Soft assert** — в отчёте и live-run помечается отдельно; в `report.json` у шага поле `softFailure`.
